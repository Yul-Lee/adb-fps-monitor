常见问题

设备未检测到

现象：设备下拉框为空，显示"未检测到设备"。

解决：

  • 确认 USB 调试已开启（设置 → 开发者选项 → USB 调试）
  • 确认 USB 线支持数据传输（非仅充电线）
  • 在终端运行 adb devices 确认设备已识别
  • 如果是首次连接，需要在设备上确认 USB 调试授权弹窗
  • 尝试重新插拔 USB 线或重启 ADB 服务：adb kill-server && adb start-server

无 FPS 数据

现象：监控已开始但 FPS 图表无数据。

解决：

  • 确认目标应用正在渲染（有动画/滚动等），停留在桌面或静态页面无法采集帧数据
  • 切换到有动画的应用（如游戏、视频、浏览器滚动）
  • 工具会自动尝试 4 种 FPS 数据源，部分应用可能不支持某些数据源

温度传感器为空

现象：温度图表无数据。

解决：

  • 部分设备的 sysfs thermal_zone 权限受限，需要 root 权限
  • 工具会自动降级到 thermalservice → battery 温度回退
  • 可在 temp_map.json 中添加自定义传感器映射

WiFi 连接速度慢

现象：数据采集延迟明显。

原因：ADB over WiFi 的每次 subprocess 调用有 100-200ms 额外延迟（USB 约 10-30ms）。

建议：优先使用 USB 连接，WiFi 连接适合无法 USB 的场景。

AOD 息屏显示异常 FPS

现象：息屏状态下显示 5-15 FPS。

原因：SFBuffFPS 无法区分 AOD（Always-On Display）Surface，会统计到息屏渲染帧。

GPU 负载/频率为空

现象：GPU 相关数据不显示。

原因：

  • 部分设备的 GPU sysfs 被 SELinux 封死，需要 root
  • 工具支持骁龙（kgsl）、Mali、PowerVR、AMD RDNA 等平台，其他平台可能无数据
